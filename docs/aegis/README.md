# Aegis Project Workspace

This directory preserves reviewed design intent and architecture evidence for Home Assistant Codex Bridge. It keeps proposed work separate from the current implementation so plans and later changes can be checked against explicit boundaries.

## Structure

- `baseline/` records dated snapshots of the implemented system.
- `specs/` contains proposed and approved behavior or architecture.
- `plans/` contains implementation plans created after a specification is approved.
- `adr/` records durable architecture decisions after implementation proves them.
- `BASELINE-GOVERNANCE.md` defines how snapshots and implementation drift are reviewed.
- `INDEX.md` lists the workspace artifacts.

Baseline snapshots are evidence, not a substitute for the code or current repository instructions. Proposed specifications do not become accepted architecture merely by being written.
